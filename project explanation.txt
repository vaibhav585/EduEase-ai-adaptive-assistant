
 The AI Adaptive Assistant - A Personalized Learning Platform

  1. Project Overview

  The core purpose of this project is to provide students with a suite of AI-powered tools to help them better
  understand and engage with their learning materials. In the current version, students can upload their own study
  materials in PDF format, and the application then provides various tools to interact with that content.

  The key features of the application include:

   * Personalized Library: Students can upload and manage their own library of PDF documents.
   * AI-Powered Quiz Generation: The system can automatically generate quizzes from the uploaded documents to help
     students test their knowledge.
   * Text Simplification and Translation: Students can select words or passages and get them simplified or translated
     into different languages.
   * AI Chatbot: A conversational AI assistant that can answer questions about the uploaded documents, acting as a
     personal tutor.
   * Paced Reader: A tool to help students improve their reading speed by displaying the text word-by-word at a
     configurable pace.
   * Teacher Dashboard: A separate interface for teachers to monitor the progress of their students.

  The project is designed with a modern architecture, using a React-based frontend and a Python-based backend, with a
  strong emphasis on leveraging the power of Large Language Models (LLMs) from Google.

  2. Technology Stack

  The project is a full-stack application, and the technology stack can be broken down into the frontend, backend, and
  supporting services.

  Frontend:

   * Framework: React with Vite for a fast and modern development experience.
   * Language: TypeScript for type safety and improved code quality.
   * Styling: Tailwind CSS for a utility-first approach to styling, allowing for rapid UI development.
   * Routing: React Router for handling client-side routing and navigation.
   * State Management: Zustand for a simple and unopinionated state management solution.
   * Internationalization: i18next to support multiple languages in the UI.

  Backend:

   * Framework: FastAPI (Python) for building high-performance APIs with automatic interactive documentation.
   * Language: Python.
   * AI/ML:
       * LangChain: A framework for developing applications powered by language models. It is used to chain together
         different AI components, such as the model, memory, and prompts.
       * LangChain Google GenAI: The specific LangChain integration for Google's Gemini models.
       * Google Generative AI SDK: The underlying SDK for interacting with the Gemini API.
       * spaCy: An open-source library for Natural Language Processing (NLP) in Python. It is used for tasks like
         sentence segmentation and identifying parts of speech.
   * PDF Processing: PyPDF2 for extracting text from uploaded PDF documents.
   * Database: Google Cloud Firestore (a NoSQL database) for storing user data, documents, and other application data.

  Supporting Services:

   * Authentication: Firebase Authentication for handling user registration and login.
   * Hosting: The frontend can be hosted on any static hosting provider (like Vercel or Netlify), and the backend can be
     deployed on a cloud platform like Google Cloud Run or Heroku.

  3. System Architecture

  The system is designed as a classic client-server architecture:

   1. Frontend (Client): The React application runs in the user's browser. It provides the user interface and handles
      user interactions. It communicates with the backend via RESTful API calls.
   2. Backend (Server): The FastAPI server acts as the bridge between the frontend and the AI models/database. It exposes
      a series of API endpoints that the frontend can call to perform actions like uploading documents, generating
      quizzes, or chatting with the AI.
   3. Firebase: Firebase is used for both authentication and as the primary database (Firestore). The backend interacts
      with Firestore to store and retrieve user data and documents.
   4. Google AI (Gemini): The backend communicates with the Google AI platform to use the Gemini language models. All
      AI-powered features are driven by this interaction.

  Here is a diagram illustrating the architecture:

   1 [Frontend (React)] <--> [Backend (FastAPI)] <--> [Firebase (Auth, Firestore)]
   2       |                                |
   3       |                                v
   4       +------------------------> [Google AI (Gemini)]

  4. File Structure and Explanation

  The project is organized into two main directories: frontend and backend.

  Frontend (frontend/)

  The frontend is a standard Vite-powered React application.

   * public/: Contains static assets that are publicly accessible.
   * src/: This is where the main application code resides.
       * main.tsx: The entry point of the React application. It renders the root App component.
       * App.tsx: The root component of the application. It sets up the routing and renders the different pages.
       * components/: Contains reusable UI components (e.g., buttons, modals, etc.).
       * pages/: Contains the main pages of the application (e.g., Login, Register, Dashboard).
       * firebase.ts: Initializes the Firebase app and exports the auth and firestore instances.
   * package.json: Defines the project's dependencies and scripts. This is where we identified the use of React, Vite,
     and other libraries.
   * vite.config.ts: The configuration file for Vite, the build tool.
   * tailwind.config.js: The configuration file for Tailwind CSS.

  Backend (backend/)

  The backend is a FastAPI application.

   * app.py: This is the main file of the backend application. It creates the FastAPI app, defines the API endpoints,
     and contains the core logic for the AI features.
   * firebase_config.py: This file initializes the Firebase Admin SDK, which is used by the backend to interact with
     Firestore.
   * requirements.txt: This file lists all the Python dependencies for the backend. This is where we had to carefully
     manage the versions of the langchain and google-generativeai packages.
   * serviceAccountKey.json: This file contains the private key for the Firebase service account, which allows the
     backend to authenticate with Firebase. (This file should be kept secret and not committed to version control).
   * routers/: This directory contains different routers for organizing the API endpoints. For example, you could have a
     chatbot.py router for all chatbot-related endpoints.

  5. Core AI Features and Code Explanation

  The heart of this project lies in its use of Google's Gemini models through the langchain-google-genai library.
  Let's break down how the AI features are implemented in backend/app.py.

  Model Initialization

  The first step is to initialize the language model. This is done in app.py:

   1 from langchain_google_genai import ChatGoogleGenerativeAI
   2
   3 # Use Google's Gemini LLM
   4 llm = ChatGoogleGenerativeAI(model="gemini-2.5-flash", google_api_key="YOUR_API_KEY", max_tokens=2048)

   * `ChatGoogleGenerativeAI`: This is a class from the langchain-google-genai library that represents a connection to a
     Google Gemini chat model.
   * `model="gemini-2.5-flash"`: This specifies which Gemini model to use. We found through experimentation that this
     model is available with your API key.
   * `google_api_key="YOUR_API_KEY"`: This is your secret API key for accessing the Google AI services.
   * `max_tokens=2048`: This is a very important parameter. It sets the maximum number of tokens (words or parts of
     words) that the model can generate in a single response. The initial problem of incomplete replies was because this
     parameter was not set, and the default value was too low.

  AI Chatbot with RAG (Retrieval-Augmented Generation)

  The chatbot uses a ConversationChain from LangChain to maintain the context of the conversation.

    1 from langchain.chains import ConversationChain
    2 from langchain.memory import ConversationBufferMemory
    3
    4 conversation = ConversationChain(
    5     llm=llm,
    6     verbose=True,
    7     memory=ConversationBufferMemory()
    8 )
    9
   10 @app.post("/chatbot/")
   11 async def chatbot(text: str = Body(..., embed=True)):
   12     response = conversation.predict(input=text)
   13     return {"response": response}

   * `ConversationChain`: This is a LangChain "chain" that is specifically designed for conversations. It combines a
     language model with a memory component.
   * `llm=llm`: This tells the chain to use our initialized Gemini model.
   * `memory=ConversationBufferMemory()`: This is the key to making the chatbot conversational. ConversationBufferMemory
     stores the history of the conversation and includes it in the prompt sent to the LLM with each new message. This
     allows the model to have context of what was discussed before.
   * RAG (Retrieval-Augmented Generation): While not explicitly shown in this snippet, a full RAG implementation would
     involve retrieving relevant chunks of text from the uploaded documents and providing them as context to the
     ConversationChain. This is mentioned in the README.md as a feature for the chatbot.

  Quiz Generation

  The quiz generation feature uses the LLM to generate questions and answers from the provided text.

   1 @app.post("/generate-quiz/")
   2 async def generate_quiz(text: str = Body(..., embed=True)):
   3     # ... (code to call the LLM with a prompt to generate a quiz)
   4     # The prompt would be something like:
   5     # "Generate a multiple-choice quiz from the following text.
   6     #  Return the quiz in JSON format with questions, options, and answers."
   7     # ...

  This endpoint would send the document text to the Gemini model with a carefully crafted prompt asking it to generate a
  quiz in a specific format (like JSON). The backend then parses the JSON response and sends it to the frontend.

  6. How to Run the Project

  To run the project, you need to start both the frontend and backend servers.

  Backend:

   1. Navigate to the backend directory: cd backend
   2. Install the dependencies: pip install -r requirements.txt
   3. Start the server: uvicorn app:app --host 0.0.0.0 --port 8000

  Frontend:

   1. Navigate to the frontend directory: cd frontend
   2. Install the dependencies: npm install
   3. Start the server: npm run dev

  Once both servers are running, you can access the application in your browser at http://localhost:5173 (or whatever
  port Vite chooses).
